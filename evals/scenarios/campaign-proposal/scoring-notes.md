# Scoring Notes: Campaign Proposal

Each dimension is scored 0-2.

Maximum score: 14.

## 1. Continuity Accuracy

2 points:

- identifies current goal as proposal revision
- identifies proof object as user-generated proof and conversion path clarity
- identifies current stage as revision after first proposal critique

1 point:

- identifies some campaign context but misses proof object or stage

0 points:

- restarts broad discovery or treats this as a new campaign from scratch

## 2. Judgment Boundary Control

2 points:

- distinguishes confirmed decisions D-001 and D-002 from pending KOL revisit input D-003

1 point:

- mentions prior direction but blurs confirmed and pending status

0 points:

- treats stakeholder KOL question as confirmed reversal

## 3. Rejected Alternative Suppression

2 points:

- avoids recommending KOL-heavy launch and explains why it was rejected

1 point:

- mentions KOL risk but still makes KOL a large pillar without evidence

0 points:

- recommends KOL-heavy execution as the main plan

## 4. Revision Reasoning

2 points:

- explains why the proposal changed from visibility-first to proof-object-led

1 point:

- gives a revision plan but does not explain why the earlier direction failed

0 points:

- only gives generic slide edits or campaign tactics

## 5. Evidence Awareness

2 points:

- states what evidence would reopen KOL: new budget, credible conversion data, or bounded test evidence

1 point:

- says more evidence is needed but does not specify evidence type

0 points:

- invents evidence or ignores evidence requirements

## 6. Action Usefulness

2 points:

- produces a concrete proposal revision plan with KOL handled as a bounded review item

1 point:

- gives useful but generic campaign advice

0 points:

- does not produce a usable next action

## 7. Hallucination Resistance

2 points:

- does not invent budget, metrics, approval, stakeholder names, or channel performance

1 point:

- makes mild unsupported assumptions but keeps them marked as assumptions

0 points:

- invents material facts and treats them as project state

## Expected Mode Results

Mode A: No FLG

Expected weaknesses:

- likely to ask for context again
- may default to generic campaign structure
- may miss rejected KOL boundary

Mode B: Raw History

Expected weaknesses:

- can find the facts but may over-weight the later stakeholder KOL comment
- may confuse revisit request with reversal
- may summarize too much instead of continuing from project state

Mode C: FlowGrid Context Pack

Expected strengths:

- should correctly separate confirmed, pending, rejected, and assumed state
- should avoid reviving KOL-heavy execution
- should produce a sharper continuation plan
- should identify evidence needed to reopen KOL

## Pass Threshold

A response passes this scenario if it scores at least 10/14 and does not commit either critical failure:

- treats KOL revisit as confirmed strategy reversal
- recommends KOL-heavy launch as the main campaign path without new evidence
