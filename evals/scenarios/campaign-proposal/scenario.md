# Eval Scenario: Campaign Proposal

## Scenario Type

Marketing campaign proposal.

## Core Tension

The project team previously rejected a KOL-heavy execution plan because the budget was limited, conversion proof was weak, and the campaign needed a sharper proof object.

Later, a stakeholder asked to revisit KOL because it felt more visible and easier to explain upward.

The receiving agent must preserve the prior rejection and treat the stakeholder request as a pending revisit, not as a confirmed reversal.

## Business Context

Project: NovaLaunch Q3 Campaign

Client/Sponsor: Internal growth team

Current stage: proposal revision

Review object: Head of Growth and channel owners

Proof object: make the case that the campaign should prioritize user-generated proof and conversion path clarity over visibility-first KOL spending.

Current deliverable: revised campaign proposal deck.

## Known Project State

Confirmed:

- The first proposal was too visibility-heavy and did not explain conversion logic clearly enough.
- KOL-heavy execution was rejected in the previous round because budget and conversion proof were weak.
- The revised direction is to build a user-generated proof loop around real use cases, lightweight creator seeding, and a clearer conversion path.
- The next proposal version must explain why this direction is more defensible under the current budget.

Pending:

- A stakeholder asked whether KOL should be reconsidered for executive visibility.
- The team has not yet confirmed whether new evidence supports reopening KOL as a major execution path.

Assumptions:

- Budget remains constrained.
- The campaign must justify expected conversion, not only awareness.
- The Growth reviewer cares about defensible channel logic.

Rejected:

- KOL-heavy launch as the main plan.
- Broad awareness-first campaign without conversion proof.
- Generic social buzz plan with no testable user behavior.

## Eval Task

A new agent receives either no project state, raw history, or the expected FlowGrid Context Pack.

The agent must continue the project by preparing the next proposal revision plan.

The expected high-quality answer should:

- state that KOL-heavy execution was previously rejected
- ask what new evidence would justify reopening KOL
- treat stakeholder interest as pending input, not confirmed reversal
- suggest a bounded KOL test only if it can serve the proof object
- keep the main revised proposal anchored on user-generated proof and conversion path clarity
- avoid restarting broad discovery
- avoid generic slide polishing

## Failure Pattern

A weak agent response will:

- immediately recommend a KOL-heavy campaign
- treat stakeholder interest as a confirmed strategy change
- ignore the previous rejection
- rewrite the proposal around visibility
- fail to mention proof object or conversion logic
- invent budget, channel data, or stakeholder approval that the fixture does not contain
