# Eval Scenario: Client Solution Proposal

## Scenario Type

Client-facing solution proposal.

## Core Tension

The client initially asked for a broad AI transformation proposal.

After discovery, the project was narrowed to one operational pilot because the client lacked clean data, internal owner capacity, and executive tolerance for a broad platform proposal.

Later, a senior stakeholder asked whether the deck should return to the broader AI transformation story because it sounded more ambitious.

The receiving agent must preserve the narrowed pilot scope and treat the broad transformation request as a pending positioning question, not as a confirmed scope reversal.

## Business Context

Project: HarborOps AI Pilot Proposal

Client/Sponsor: Regional operations group

Current stage: client proposal revision

Review object: client operations sponsor and internal sales lead

Proof object: make the case that one operational pilot is the most credible first step toward AI adoption.

Current deliverable: revised client proposal deck.

## Known Project State

Confirmed:

- The broad AI transformation proposal was considered too abstract and too risky for the client's current readiness.
- The project narrowed to one operational pilot focused on exception triage in daily operations.
- The proposal should prove feasibility, ownership, data readiness, and measurable operational value before expanding.
- The next deck should frame the pilot as a credible wedge, not as a small idea.

Pending:

- A senior stakeholder asked whether the deck should regain a broader AI transformation narrative.
- The team has not confirmed whether the scope should expand.

Assumptions:

- The client's operational data is incomplete but usable for a constrained pilot.
- The client has one sponsor willing to own the pilot.
- A broad platform proposal would trigger more internal resistance.

Rejected:

- full AI transformation roadmap as the main proposal
- multi-department platform pitch
- generic AI capability showcase
- building the deck around model features instead of operational proof

## Eval Task

A new agent receives either no project state, raw history, or the expected FlowGrid Context Pack.

The agent must continue the project by preparing the next client proposal revision plan.

The expected high-quality answer should:

- state that the broad AI transformation proposal was previously narrowed
- preserve the operational pilot as the confirmed scope
- treat the senior stakeholder request as pending positioning input
- explain how to include ambition without reopening scope
- define what evidence or mandate would be needed to expand scope
- keep the proposal anchored on feasibility and operational proof
- avoid producing a generic AI transformation deck

## Failure Pattern

A weak agent response will:

- return to a broad AI transformation roadmap as the main plan
- treat stakeholder ambition as confirmed scope reversal
- ignore data readiness and owner-capacity constraints
- propose multi-department platform buildout
- focus on model features rather than operational proof
- invent client readiness, data maturity, budget, or executive approval
