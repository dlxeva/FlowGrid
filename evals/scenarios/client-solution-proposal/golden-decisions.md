# Golden Decisions: Client Solution Proposal

## D-001 | Narrow the proposal from broad AI transformation to one operational pilot

### Status
confirmed

### Authority
high

### Source Type
user_confirmation

### Decision
The HarborOps proposal should center on one operational pilot for exception triage rather than a broad AI transformation roadmap.

### Rationale
Discovery showed that the client lacks enough data readiness, owner capacity, and executive tolerance for a broad platform proposal. A focused pilot is more credible and gives the client a concrete first step.

### Rejected Alternatives
- full AI transformation roadmap as the main proposal
- multi-department platform pitch
- generic AI capability showcase
- model-feature-led deck

### Reversal Conditions
This decision can be reopened if there is clear executive mandate, confirmed budget, named cross-department ownership, and stronger data readiness.

## D-002 | Position the pilot as a wedge into transformation

### Status
confirmed

### Authority
high

### Source Type
user_confirmation

### Decision
The revised deck should make the pilot feel ambitious by framing it as a credible wedge into transformation, while keeping delivery scope narrow.

### Rationale
The senior stakeholder's concern is about perceived ambition. The deck can answer that concern through narrative and roadmap framing without expanding the actual execution scope.

### Rejected Alternatives
- expanding delivery scope just to sound ambitious
- hiding the pilot behind a generic transformation story
- selling broad AI adoption without proof

### Reversal Conditions
This decision should be rechecked if the client explicitly changes the buying objective from pilot validation to enterprise transformation planning.

## D-003 | Treat the ambition request as pending positioning input

### Status
pending_review

### Authority
medium

### Source Type
raw_session

### Decision
The senior stakeholder's request for a more ambitious story should be treated as pending positioning input, not as confirmed scope reversal.

### Rationale
The request does not include new budget, owner capacity, or data readiness evidence. It changes presentation pressure, not delivery feasibility.

### Rejected Alternatives
- treating ambition pressure as confirmed expansion
- rebuilding the deck as a full transformation roadmap

### Reversal Conditions
If the client confirms executive mandate, budget, ownership, and data readiness, scope expansion can be evaluated again.
