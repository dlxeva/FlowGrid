# Golden Decisions: Operations Mechanism Design

## D-001 | Simplify the activity mechanism for frontline execution

### Status
confirmed

### Authority
high

### Source Type
user_confirmation

### Decision
The activity mechanism should be simplified from a multi-layer gamified system into a lightweight participation loop.

### Rationale
The original mechanism created too much explanation, tracking, moderation, and settlement complexity. Frontline moderators need a rule system they can explain and operate reliably.

### Rejected Alternatives
- multi-track task system
- weekly ranking as the main incentive
- badge economy
- bonus task stack
- complex point settlement

### Reversal Conditions
This decision can be reopened if the team has evidence that the added mechanism can be explained clearly, moderated reliably, settled without disputes, and improve quality participation.

## D-002 | Anchor the revised mechanism on one core task and one simple reward rule

### Status
confirmed

### Authority
high

### Source Type
user_confirmation

### Decision
The revised mechanism should use one core task, one proof submission, one weekly showcase, and one simple reward rule.

### Rationale
This keeps the activity understandable to users and operable for moderators while preserving a basic motivation loop.

### Rejected Alternatives
- adding multiple participation tracks
- designing a full gamified economy
- optimizing for perceived richness over operational reliability

### Reversal Conditions
This decision should be rechecked if the operations team gains more moderation capacity or if a test shows that extra mechanics improve quality participation without creating disputes.

## D-003 | Treat points/ranking restoration as pending engagement input

### Status
pending_review

### Authority
medium

### Source Type
raw_session

### Decision
A stakeholder asked whether points and ranking should be restored for engagement, but this does not yet reverse the simplified mechanism direction.

### Rationale
The request signals concern about engagement, but does not answer the execution-cost problem that caused the simplification.

### Rejected Alternatives
- restoring points and ranking as the default mechanism
- making ranking the main incentive without moderation evidence

### Reversal Conditions
If a bounded test proves that points or ranking increase quality participation without overwhelming moderators, limited reintroduction can be evaluated.
