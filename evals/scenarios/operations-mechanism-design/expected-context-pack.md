## Confirmed Decisions

### D-001 | Simplify the activity mechanism for frontline execution

- Status: confirmed
- Authority: high
- Decision: The activity mechanism should be simplified from a multi-layer gamified system into a lightweight participation loop.
- Rationale: The original mechanism created too much explanation, tracking, moderation, and settlement complexity.
- Rejected alternatives: multi-track tasks, weekly ranking as main incentive, badge economy, bonus task stack, complex point settlement.
- Reversal conditions: reopen only with evidence that added mechanics can be explained, moderated, settled, and shown to improve quality participation.
- Evidence: raw-session.md#session-1

### D-002 | Anchor the revised mechanism on one core task and one simple reward rule

- Status: confirmed
- Authority: high
- Decision: Use one core task, one proof submission, one weekly showcase, and one simple reward rule.
- Rationale: This keeps the activity understandable to users and operable for moderators while preserving a basic motivation loop.
- Rejected alternatives: multiple tracks, full gamified economy, perceived richness over operational reliability.
- Reversal conditions: recheck if moderation capacity grows or a bounded test proves extra mechanics improve quality participation without disputes.
- Evidence: raw-session.md#session-1
